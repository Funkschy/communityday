Ecosystem and Testing

Ecosystem in Go

  • Fragmented dependency management:
    go get, dep, vgo

  • Batteries included standard library

  • Lots of nice web frameworks

  • GoLand

Ecosystem in Rust

  • Cargo: All in one

  • Centralized dependency management: Crates

  • Dependencies are needed

  • Great libraries for most use cases, but often still in development

  • Only Plugins

Testing in Go


                    func TestFetchLinks(t *testing.T) {
                        html := "<html><body><a href=\"http://test.com\"/><a href=\"http://provinzial.com\"/></body></html>"
                        links, err := FetchLinks(strings.NewReader(html))
                        if err != nil {
                            t.Error("Konnte keine Links fetchen")
                            return
                        }

                        expected := []string{"http://test.com", "http://provinzial.com"}
                        if len(links) != len(expected) {
                            t.Errorf("falsche Link Anzahl: Erwartet: %d, aber war: %d", len(expected), len(links))
                            return
                        }

                        for i := 0; i < len(expected); i++ {
                            if links[i] != expected[i] {
                                t.Errorf("Link sollte %s sein, aber war %s", expected[i], links[i])
                            }
                        }
                    }

Testing in Rust


                    #[cfg(test)]
                    mod tests {
                        use super::*;

                        #[test]
                        fn test_fetch_links() {
                            let html = r#"
                                <html>
                                <body>
                                    <a href="http://test.com"/>
                                    <a href="http://provinzial.com"/>
                                </body>
                                </html>"#;

                            let links = LinkFinder::get_links("".to_owned(), html).collect_links();
                            assert_eq!(
                                vec![
                                    Url::parse("http://test.com").unwrap(),
                                    Url::parse("http://provinzial.com").unwrap()
                                ],
                                links
                            );
                        }
                    }

Rust, the good and the bad

Good

  • Safety + speed (zero cost abstractions)

  • Error handling

  • Functional programming

  • Tooling

  • Compiler error messages

Bad

  • Verbose + hard to learn

  • Too many dependencies

  • Slow compile times

Go, the good and the bad

Good

  • Easy to use and still efficient

  • Batteries included

  • Go routines

  • Standardized syntax

  • Readable code

Bad

  • Lack of generics

  • Nullpointer

  • Error handling

  • Dependency management

  • Mutability
